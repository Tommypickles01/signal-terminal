How signals fire: a green BUY dot prints when momentum confluence flips up (RSI exiting oversold, price reclaiming EMA21, EMA9 crossing up, or a volume-spike green candle) and a red SELL dot prints on the mirror conditions. It needs 2+ confirmations and alternates buy→sell→buy so you don't get spammed. Fib levels auto-draw from the swing high/low in view.

Are the dots real? Yes — they're computed live from actual candle data, not random. They're lagging momentum signals (TA), not predictions. The newest forming candle can repaint until it closes, then its dot locks in. LIVE updates price every 6s + candles every 60s. MCAP rescales the chart to market cap. DEX ⧉ swaps in DexScreener's true tick chart.

Indicator buttons: VOL = volume bars; BOLLINGER = 20-period SMA ±2σ band (volatility squeeze/expansion); FIB = auto Fibonacci off the swing high/low in view; SUPPORT/RES = auto levels clustered from swing pivots (thicker = more touches); PROJECTION = a linear-regression trend extended forward with a widening cone. The projection is a math extrapolation of the current trend, not a prediction — it shows where price goes if the trend holds. WAVETREND opens a Market Cipher-B style oscillator pane: the cyan wave + blue signal line, with green dots on bullish crosses (esp. below the −60 oversold band) and red dots on bearish crosses (above +60). Everything recomputes on every live tick and rescales with PRICE/MCAP.

Heads up — this is not financial advice. Memecoins are extremely high risk; indicators describe momentum, they don't predict rugs, LP pulls, or dev wallets. Size accordingly.

Cloud sync (Supabase) — optional

How to turn on cloud sync (free):
1. Make a free project at supabase.com → Project Settings → API → copy the Project URL and anon public key into the fields above.
2. In the Supabase SQL editor, paste & run this:
create table if not exists app_state ( user_id uuid primary key references auth.users(id) on delete cascade, data jsonb not null default '{}'::jsonb, updated_at timestamptz not null default now() ); alter table app_state enable row level security; create policy "own row" on app_state for all using (auth.uid() = user_id) with check (auth.uid() = user_id);
3. Put your email above, hit SEND MAGIC LINK, click the link in your inbox, come back and hit SYNC NOW. Your account, trades, and signal memory now follow you across devices. The anon key is safe to keep in this public file — Row-Level Security means each user only ever sees their own row.
